---

Bits are small, self-contained notes, code snippets, short writeups, or interactive demos I find worth keeping around.

<br />

No long essays. No long tutorials. Just the thing itself: a working piece of code, a quick insight, or a pattern I want
to remember (and maybe you'll find useful too).

<br />

There might be some exceptions to the "short" rule, for example for some storytelling, but i'll try my best to keep bits
concise and to the point.
